Вот полный список того, что нужно доработать для продакшена, по приоритету:

Критические (безопасность и стабильность)
TLS/SSL — сейчас WebSocket работает без шифрования. Нужен HTTPS/WSS (через reverse proxy типа nginx/caddy или встроенный TLS).

Rate limiting — нет защиты от brute-force и DDoS. Нужны лимиты на auth.sendCode, auth.signIn, и на все RPC-вызовы.

Валидация входных данных — нет проверки размера сообщений, длины имён, пустых строк, невалидных peerKey. Потенциальные crash/injection.

Auth-коды в логе — auth.sendCode печатает код в консоль. В проде нужна интеграция с SMS/email сервисом или хотя бы скрытие из логов.

Хардкоженный RSA-ключ — один и тот же для всех инстансов. Нужна генерация уникальных ключей при установке.

Сессии без TTL — сессии живут вечно, нет механизма истечения/принудительного выхода (account.getAuthorizations — стаб).

Высокий приоритет (ключевые фичи)
Отправка медиа (messages.sendMedia) — сейчас возвращает MEDIA_INVALID. Файлы хранятся только в памяти (upload.saveFilePart), теряются при рестарте. Нужна:

Таблица documents/photos в SQLite
Персистентное хранение файлов на диск
Привязка media к сообщениям
Поиск сообщений (messages.search) — возвращает пустой вектор. Нужен FTS5-индекс в SQLite по тексту сообщений.

Группы/каналы — полностью отсутствуют:

Таблица chat_participants (member list, roles)
Права/модерация (admin, ban, mute)
channels.createChannel, messages.createChat
Рассылка обновлений всем участникам
Реакции — messages.sendReaction — стаб, нет таблицы для хранения реакций.

Пересланные сообщения — messages.forwardMessages работает, но не хранит метаданные (fwd_from: original author, date, channel).

Черновики (drafts) — messages.saveDraft возвращает true, но ничего не сохраняет.

Средний приоритет (улучшение UX)
2FA (двухфакторная аутентификация) — account.getPassword возвращает "нет пароля". Нужна реализация SRP-протокола.

Превью ссылок (messages.getWebPagePreview) — стаб. Нужен HTTP-fetch + парсинг OpenGraph.

Закреплённые сообщения — messages.getPinnedDialogs возвращает пустой список. Нужно хранение pin-статуса.

Entities в сообщениях — bold, italic, ссылки, mentions не парсятся и не хранятся при отправке.

Статус онлайн — persistence — account.updateStatus бродкастит, но не сохраняет lastSeenAt в БД.

Папки (dialog filters) — messages.getDialogFilters возвращает пустой вектор. Нужна таблица + CRUD.

Запланированные сообщения — messages.getScheduledHistory — стаб. Нужна таблица + cron/timer.

Низкий приоритет (полнота API)
Стикеры/GIF — все sticker-хендлеры возвращают NotModified. Нужна интеграция с хранилищем стикерпаков.

Голосовые/видеозвонки — нет сигнального протокола. Нужен WebRTC relay или Telegram-совместимый VoIP.

Секретные чаты — нет E2E-слоя (отдельный от MTProto 2.0).

Stories — все stories.* возвращают пустые ответы.

Боты/inline-запросы — не обрабатываются.

Push-нотификации — нет механизма push для мобильных клиентов (нужен FCM/APNs).

Настройки приватности — account.getPrivacy возвращает пустые правила.

Инфраструктура
Мониторинг/метрики — нет логов структурированных, нет Prometheus/Grafana, нет health-check endpoint.
Горизонтальное масштабирование — один процесс, один SQLite. Для нагрузки нужен PostgreSQL + кластер.

Бэкапы БД — нет механизма автоматического бэкапа SQLite.

Graceful shutdown — нужно корректное закрытие WebSocket-соединений и flush pending writes.

Тесты — минимальное покрытие. Нужны E2E-тесты на весь auth + messaging flow.

Итого: ~15 полноценных хендлеров из ~130, файлы теряются при рестарте, нет групп/каналов, нет медиа, нет поиска. Для MVP с приватным P2P-чатом ближе всего пункты 1-8.
